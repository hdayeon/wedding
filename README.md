# 모바일 청첩장

빌드 도구 없는 정적 페이지. `index.html` 한 파일이 전부입니다.

**배포 주소:** https://hdayeon.github.io/wedding/

## 수정하기

- **이름·날짜·연락처·식장 주소·카카오 키** → `index.html` 안 `CONFIG` 객체
- **인사말 문구, 혼주, 계좌번호** → `index.html` 본문 HTML
- **사진** → `images/` 폴더 ([안내](images/README.md))

## 카카오톡 공유, 두 가지가 다릅니다

헷갈리기 쉬운데 별개의 기능이고, **둘 다 설정해야 합니다.**

### 1. 링크를 붙여넣었을 때 (OG 태그)

주소를 복사해서 카톡 대화창에 붙여넣으면 카카오 크롤러가 페이지를 읽어
미리보기 카드를 만듭니다. `<head>`의 `og:` 태그가 그 내용입니다.

- 이미지가 **가로형으로 작게** 나오고, **버튼은 붙지 않습니다**
- `images/og.jpg` (1200×630) 필요
- 키 발급 불필요

### 2. 공유 버튼을 눌렀을 때 (Kakao SDK, Feed 템플릿)

페이지의 노란 버튼을 누르면 카톡 공유 시트가 열리고, **큰 세로 이미지 +
버튼 2개**가 달린 메시지가 전송됩니다. 시중 청첩장 서비스가 쓰는 방식이
이겁니다. OG 태그와 무관하며, `Kakao.Share.sendDefault()`로 보냅니다.

- `images/share.jpg` (640×852, **세로 3:4**) 필요 — 비율이 다르면 작게 나옵니다
- 카카오 JavaScript 키 필요 (아래 참고)

## 카카오 공유 버튼 켜기

1. https://developers.kakao.com → 내 애플리케이션 → 애플리케이션 추가하기
2. **앱 키 → JavaScript 키** 복사 → `index.html`의 `CONFIG.kakaoJsKey`에 붙여넣기
3. **플랫폼 → Web → 사이트 도메인**에 `https://hdayeon.github.io` 등록

카카오 로그인 활성화는 **필요 없습니다.** 공유하기는 비로그인으로 동작합니다.
키를 넣지 않으면 카카오 버튼은 자동으로 숨겨지고 "링크 복사"만 표시됩니다.

> JavaScript 키는 브라우저에 노출되는 값이라 공개되어도 괜찮습니다.
> 3번의 도메인 등록이 그 키를 쓸 수 있는 사이트를 제한해 줍니다.
> 반면 **REST API 키·Admin 키는 절대 이 파일에 넣으면 안 됩니다.**

## 로컬에서 보기

```sh
python3 -m http.server 8877
```

`file://`로 직접 열면 계좌 복사(Clipboard API)가 막히므로 서버로 여는 편이 정확합니다.
카카오 공유 버튼은 `localhost`가 도메인 등록이 안 되어 있어 실기기 테스트가 필요합니다.

## 배포

`main` 브랜치에 push하면 1~2분 뒤 자동 반영됩니다.
